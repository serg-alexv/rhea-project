Rhea Tribunal API

Send one prompt. Get answers from 3-7 AI models. Receive structured consensus analysis with agreement scores, stance detection, and divergence maps.

Get Started
# One API call. Multiple models. Structured consensus.
curl -X POST https://api.rhea-tribunal.dev/tribunal \
  -H "X-API-Key: your-key" \
  -d '{"prompt": "Is intermittent fasting safe for diabetics?", "k": 5}'

# Response:
{
  "agreement_score": 0.72,
  "confidence": 0.81,
  "consensus": "4/5 models agree: generally safe for Type 2 with monitoring...",
  "divergence_points": ["Gemini flags contraindication for insulin-dependent"],
  "stance_summary": {"affirmative": 3, "qualified": 2}
}

Why Tribunal?

Cross-Model Validation

Every LLM hallucinates differently. When 5 models agree, confidence goes up. When they diverge, you know where the uncertainty lives.

Structured Output

Not just "5 models responded." You get agreement scores, stance analysis, pairwise similarity, and specific divergence points.

ICE Iterative Consensus

Models critique each other in rounds until convergence. Based on ICE (2025): +7-15% accuracy over single-model inference.

Cost-Aware Routing

4 cost tiers. Cheap by default. Expensive only when you need it. Every call logged for budget tracking.

Three Analysis Levels

L1

Local Analysis

TF-IDF similarity + stance detection + claim overlap. Pure math, no extra API calls. Included in every request.

Cost: $0 extra (included)
L2

Chairman Council

A chairman model reads all responses and synthesizes a definitive answer. Resolves conflicts, merges best insights. Karpathy Council pattern.

Cost: +1 API call
L3

ICE Iterative

Models see each other's answers and revise. Multiple critique rounds until convergence detected. Then chairman synthesis. Maximum accuracy.

Cost: +k x rounds + 1 API calls
