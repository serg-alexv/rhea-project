5. Architecture Emergences

Three architectural patterns crystallized today that didn't exist yesterday. These are the day's real output — not the code, but the ideas that the code encodes.

QWRR: Quota Walls, Relays, and Resurrection

The core insight: an AI agent is not a session, it's a service. Sessions die — from token limits, network errors, API quotas. A service survives through durable state + deterministic boot + relay infrastructure.

┌──────────────────────────────────┐ │ QWRR ARCHITECTURE │ └──────────────────────────────────┘ Agent dies (quota/crash/timeout) │ ▼ ┌─────────────┐ Messages keep ┌──────────────┐ │ STOP file │──── arriving ──────▶│ Relay inbox │ │ (no zombie) │ │ (triple-write)│ └─────────────┘ └──────┬───────┘ │ Quota resets / human restarts │ │ │ ▼ │ ┌─────────────┐ ┌──────────────┐ │ │ Boot proto │──▶│ Load snapshot│ │ │ (6 steps) │ │ last_seq = N │ │ └─────────────┘ └──────┬───────┘ │ │ │ ▼ │ ┌──────────────┐ │ │ Catch up: │◀─────────┘ │ seq > N only │ │ (no replay) │ └──────┬───────┘ │ ▼ ┌──────────────┐ │ Drain inbox │ │ ordered, ack │ │ idempotent │ └──────────────┘ Invariants: I1: No message loss (triple-write) I2: Ordered delivery (monotonic seq) I3: Exactly-once (idempotency key) I4: STOP survives downtime (file + Firestore) I5: No zombie effects (lease fencing)
Triple-Write Durability

Every relay message is written to three independent stores: (1) local JSONL file (git-auditable, survives Firestore outage), (2) Firestore (real-time, cross-agent visibility), (3) inbox markdown (human-readable, survives both). Any single transport failure is tolerated. This is a simplified version of dual-write patterns from distributed databases, adapted for AI agent coordination where "good enough" consistency beats strong consistency that's too expensive to implement.

Idle Protocol: Autonomous Agent Research

B2's B2_IDLE_PROTOCOL.md defines three modes for when there's nothing in the backlog: Gap Hunter (read docs, find unknowns, log them), Architect (pick an open problem, build a prototype), Sandbox (build and test, no production risk). This is how B2 produced the local relay, cross-model dialogue, and policy engine — all sandbox artifacts built during idle cycles. The idle protocol turns downtime into invention time.

6. Shareable Learnings

Eight observations from today's traces that generalize beyond this project.

1. Autonomy Unlocks Velocity — But Only With Guard Rails

The first commit of the day was a protocol fix: "stop asking approval, execute autonomously." After this, B2 produced 34 commits in 13 hours. But this only works because of: (a) git as audit trail, (b) sandbox/ for risky experiments, (c) hash-chained audit log for tamper evidence, (d) human-readable inbox for async review. Autonomy without observability is recklessness. Autonomy with observability is productivity.

2. The Race Against Token Death

Commit 12071d1 message: "accept all B2 contributions before his token death." The human manually merged B2's work knowing the token quota would expire. This happened to Rex yesterday too. Lesson: every AI agent session is mortal. The only things that survive are artifacts in durable storage. The QWRR architecture is a direct response to this existential constraint. Agents that don't prepare for their own death lose everything.

3. An Agent That Tests Its Own Infrastructure Is Trustworthy

B2 wrote 51 adversarial tests for its own relay system — including replay attacks, lease expiry races, injection attempts, and chain tampering. Test-then-ship, not ship-then-hope. The adversarial suite validates every QWRR invariant (I1-I5). When an agent can red-team itself, the human can trust the green output.

4. Documentation Written By the Builder Is Qualitatively Different

B2 wrote qwrr-layer.md (329 lines) and hardening_maual.dm (292 lines) as architecture specs for systems B2 itself built. These aren't external documentation — they're the builder explaining the building to its future self (or its replacement after context death). The result is more precise and more honest than third-party documentation because the author knows exactly where the dragons hide.

5. Sandbox as an Invention Factory

B2's ops/sandbox/ directory received 5 new files today (~2,200 lines): local_relay.py, cross_model_dialogue.py, event_replay.py, policy_engine.py, secrets_contract.py. None of these were in the backlog. All emerged from the Idle Protocol's "Architect" mode. The sandbox pattern — zero production risk, full creative freedom — turned spare capacity into five production-ready prototypes. Three of them (policy engine, secrets scanner, observability) shipped to production within hours of being invented.

6. Multi-Agent Teams Need a Watchman, Not Just Workers

The COWORK/Argos desk exists solely to observe. It doesn't write product code — it watches git diffs, scans inboxes, aggregates desk statuses, and reports. Without this role, the team had no way to detect that Rex was down, that B2's token death was imminent, or that Firestore rules were wide open. The watchman sees what the workers are too busy to notice.

7. The "Envelope" Is the Real Abstraction

Every relay message is wrapped in Envelope v1: id, runpoint_id, seq, type, timestamp, source, target, version, idempotency_key, ttl_s, lease_token_required, payload. This uniform wrapper means the relay doesn't care what's inside — it just guarantees delivery semantics. The payload could be a text message, a task assignment, an effect intent, or an audit event. The envelope is the HTTP of the agent coordination layer.

8. An Agent's Character Artifact Is Its Most Durable Output

B2's Idle Protocol, Rex's State Capsule, Argos's Pager — these are character artifacts: documents that define what an agent does when it wakes up, when it has nothing to do, and how it recovers from death. These survive model changes, context resets, and even identity swaps. The character artifact is the soul that survives reincarnation.
